ECMAScript 2015

ES2015 introduced two important new JS keywords let and const.

These two keywords provide Block Scope variables (and constants) in JS.

Before ES2015 JS had only two types of scope: Global Scope and Function Scope.

Global Scope

Variables declared Globally (outside any function) have Global Scope.

var countryName = "Czech Rep.";
// code here can use countryName
function myFunction() {
// code here can also use countryName
}

Global variables can be accessed from anywhere in a JS program.

Function Scope

Variables declared Locally (inside a function) have Function Scope.

// code here cannot use countryName
function myFunction() {
var countryName = "Czech Rep.";
// code here can use countryName
}
// code here cannot use countryName

Local variables can only be accessed from inside the function where they are declared.

JS Block

Variables declared with the var keyword cannot have Block Scope.

Variables declared inside a block {} can be accessed from outside the block.

{
var x = 5;
}
// x can be used here

Variables declared with the let keyword

Before ES2015 JS didn't have Block Scope.

Variables declared inside a block {} canot be accessed from outside the block:

{
let x = 5;
}
// x cannot be used here

Redeclaring Variables

Redeclaring variable using the var keyword can impose problems.

Redeclaring a variable inside a block will also redeclare the variable outside the block:

var x = 3;
// Here x is 3
{
var x = 11;
// Here x is 11
}
Here x is 11

Redeclaring a variable using the let keyword can solve this problem.

Redeclaring a variable inside a block will not redeclare the variable outside the block.

var x = 3;
// Here x is 3
{
let x = 11;
// Here x is 11
}
Here x is 3

Loop Scope

Using var in a loop:

var i = 7;
for (var i = 0; i < 10; i++) {
// some statements
}
// Here i is 10

Using let in a loop:

var i = 7;
for (let i = 0; i < 10; i++) {
// some statements
}
// Here i is 7

Function Scope

Variables declared with var and let are quite similar when declared inside a function. The will both have Function Scope.

function myFunction() {
var countryName = 'Iceland'; // Function Scope
}
function myFunction() {
let countryName = 'Iceland'; // Function Scope
}

Variables declared with var and let are similar when declared outside a block. They both have Global Scope:

Global Variables in HTML

With JS the gloabl scope is the JS environment.

In HTML, the global scope is the window object.

Global variables defined with the var keyword belong to the window object.

Gloabl variables defined with the let keyword do not belong to the window object

Redeclaring

Redeclaring a JS variable with var is allowed anywhere in the program

Redeclaring a var with let in the same scope or in the same block is not allowed.

Hoisting

Variables defined with var are hoisted to the top and can be initialized at any time, i.e. we can use the variable before it is declred.

Varibales defined with let are hoisted to the top of the block, but not initialized, i.e. the block of code is aware of the variable, but it cannot be used until it has been declared.

Using a let variable before it is declared will result in a ReferenceError.

The variable ios in a "temporal dead zone" from the start of the block until is is declared
