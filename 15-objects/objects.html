JS Objects

Objects are variables, but objects can contain many values. The values are written as name:value pairs.

Object Definition

We define a JS object with an object literal:

var person = {
firstName:"Hans",
lastName:"Mustermann",
age:60,
eyeColor:"blue"}

Object Properties

The name:value pairs in JS objects are called properties.

Property Property Value
firstName Hans

Accessing Object Properties

We can access object properties in two ways:

  • Dot Notation
  • Bracket Notation
objectName.propertyName
objectName["propertyName"]

Object Methods

Objects can also have methods. Methods ae actions performed on objects. Methods are stored in properties as function definitions.

Property Property Value
fullName function() {return this.firstName+ "" +this.lastName;}

The this Keyword

In a function definition, this refers to the owner of the function.

This means the object.

Accessing Object Methods

We access object methods with:

objectName.methodName()

Do Not Declare Strings, Numbers and Booleans as Objects

When a JS variable is declared with the keyword new the variable is created as object:

var x = new String(); // Declares x as a String object
var y = new Number();
var z = new Boolean();

Try to avoid String, Number and Boolean objects. They complicate code and slow down exection speed.
